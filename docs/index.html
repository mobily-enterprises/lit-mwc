Powerful, themeable, enhanced native elements

The Platform Elements (TPE) is not yet another custom implementation. It is native elements, enhanced. TPE form controls are 100% customizable, and still behave like you would expect from native elements, while also providing additional capabilities.

From this...

<label for="native-input"><span>Native Input (lowercase only)</span></label>
<input id="native-input" pattern="[a-z]*" />
<input type="checkbox" id="native-checkbox" />
<label for="native-checkbox"><span>Native Checkbox</span></label>

...to this!

<nn-input-text label="TPE Input (lowercase only)" pattern="[a-z]*"></nn-input-text>
<nn-input-checkbox label="TPE Checkbox"></nn-input-checkbox>
In this example, the pattern attribute isn’t handled by TPE. Every attribute and every property is mirrored into the contained native element. TPE elements don’t need a manual, since they are native ones.

The element collection

Native elements are wrapped inside Custom Elements that reflect presence and values for all native properties and attributes. The only differences, for the form control elements, are the embedded labels and the fact that input types all have their own element, like nn-input-email and nn-input-progress.

These elements are what we call Nearly Native (hence the nn- prefix). We have also come up with two additional categories of elements that can help developers and design build great apps. They are Enhaced Native (en-) and Extra Elements (ee-).

You can learn more in the documentation section, but here is a brief summary of how they compare:

NN ELEMENTS

Look and behave like native elements by default.

EN ELEMENTS

Useful features and capabilities added to native elements.

EE ELEMENTS

Common UI patterns that have no native counterparts.

Fully customizable

All elements in the collection can be used directly or extended by Developers to include more functionality needed in your project. TPE has a robust theming solution that makes it possible to implement an entire design system as an easy to build add-on package. It also allows for the use of simple, plain CSS, while still taking advantage of shadow DOM encapsulation.
